Free · 5-minute extract

Pull your overdue list in 5 minutes.

Step-by-step for the four most common PMS systems. No SQL, no support tickets. Click your software below.

1

Open the Letter Merge

In Dentrix, the overdue-patient extract lives under the Letter Merge tool — don't look in Reports.

Office Manager › Letters & Custom Lists › Inactive Patient
2

Set the date filter

Set "Last Visit" range: 6 to 24 months ago. This captures the sweet spot — patients who've drifted but aren't totally gone.

Optional: add a second filter for "Missed Appointments > 0" to catch broken-appointment patients on the same pull.
3

Pick the merge fields

These are the fields you want in your CSV export:

First NameRequired
Last NameRequired
Mobile PhoneRequired
EmailRequired
Last VisitRequired
Preferred Contact
Insurance Plan
4

Export to CSV

Click Export to File → choose CSV. Dentrix saves to C:\DENTRIX\DOC\ by default.

That's it. One CSV. Nothing else needed on your side.
1

Open Query Generator

In Eaglesoft, the overdue-patient pull lives in the Query tool — not in Reports.

Eaglesoft › Lists & Queries › Patient Query
2

Build the filter

Add two conditions:

Last Visit Date6–24 mo
StatusActive
Pro tip: exclude "Do Not Contact" status patients at the query level — cleaner list, no manual scrub.
3

Select output fields

Right-panel checkboxes. Pick the same fields Dentrix needs:

First NameRequired
Last NameRequired
MobileRequired
EmailRequired
Last Appt DateRequired
SMS Consent
4

Run → Export

Click Run Query → Export → choose CSV. Saves to Eaglesoft's Output folder.

1

Open User Query

Open Dental's SQL-free report builder handles this cleanly.

Reports › Standard › User Query
2

Use the preset

Click Load Favorites → find "Patients Not Seen 6 Months". Clone it. Edit date range to your window.

If you're comfortable with SQL: a custom query works faster — filter on patient.DateLastVisit between your dates, joined on patient.PatStatus = 0.
3

Export

Right-click on the result grid → Export → choose CSV. That's it.

4

Optional: also pull broken appointments

Second query: filter appointment.AptStatus = 5 (Broken) within last 90 days. Merge the two CSVs.

1

Open the Dashboard

Curve is cloud-first and the overdue list is a built-in dashboard widget.

Hygiene › Recall Management › Due / Past Due
2

Filter by "Overdue"

Top-right filter pill: click Overdue. Set your range (6–24 months). Curve auto-updates the list.

Important: toggle "Include Inactive" off unless you specifically want dormant patients too — the default is active-only.
3

Export CSV

Three-dot menu (top right) → Export CSV. Curve emails the file to the logged-in user's email within ~2 minutes.

4

Heads up

Curve exports don't include SMS consent status by default. Either pull that from the patient profile view separately, or just assume not-consented and filter during outreach.
